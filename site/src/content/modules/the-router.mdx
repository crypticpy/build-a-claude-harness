import Fact from '../../components/heva/Fact.astro';
import EvidencePin from '../../components/heva/EvidencePin.astro';

Five lifecycle events are wired, but they do not run five different programs. Each
subscription in the settings file invokes the same entry point and hands it one word:
the event it is reacting to. The reaction lives inside that one script.

<Fact>One dispatcher serves every wired event. The event name arrives as a command-line argument and selects the branch.</Fact>

So the dispatcher is a switch. It reads the argument, and for `post-edit` it loads the
formatter and the logger; for `precompact` it loads the summarizer; for `session-start`
it builds the snapshot. Nothing else loads. A module is imported only on the event that
needs it, which keeps the common path cheap and keeps each module ignorant of the others.

<EvidencePin file="reference/hooks/unified/unified-hook.mjs">
The dispatcher branches on the event argument and lazy-loads the matching module. Add a
new behavior by adding a branch, not by registering a new script in the settings file.
</EvidencePin>

The payoff is in the next room. Because everything funnels through one script, the harness
can wrap that single funnel in one safety net, and every event is covered by it at once.
