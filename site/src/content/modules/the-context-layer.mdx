The hooks are one half of the harness. The other half is a separate program: a small server
that answers questions about your code. It speaks JSON-RPC over stdin and stdout, it is
written by hand with no SDK, and it offers four tools: a file summary, a consumer lookup,
a search over saved lessons, and a way to write a new lesson back.

<Fact>Four tools, one hand-rolled JSON-RPC loop, and a strict rule: a single bad message must never kill it.</Fact>

The loop reads a message, runs the requested tool, and writes a reply, forever. The danger
is obvious: this process lives for the whole session, so if one malformed message could
crash it, every tool would vanish until you restarted. So the loop wraps each message. A
parse error becomes an error reply. An unknown method becomes an error reply. The loop reads
the next message either way. Fail-silent here is not politeness; it is survival, because the
cost of dying is measured in the whole rest of the session.
