You have taken the whole harness apart. Now assemble the answer to the question the descent
was built around: how does it run for hours without you watching?

<Fact>Autonomy is containment, not speed. Each layer removes one reason you would have to stay in the loop.</Fact>

Look back at what each layer bought. Fail-silent means a broken hook degrades quietly instead
of halting your turn, so it can run unattended without stranding you. Memory means a
compaction in the third hour does not erase the first two, so it stays oriented. The gate
means the few genuinely dangerous actions are stopped, so a long run cannot quietly do
something irreversible. Verification means a turn that made several edits gets nudged to check
its own work before calling it done. None of these makes the agent smarter. Each one removes
a reason you would otherwise have to sit and watch.

<EvidencePin file="reference/settings.template.json">
The wired event set is the whole apparatus: SessionStart, UserPromptSubmit, PostToolUse,
PreCompact, Stop. Each one carries the modules that make unattended operation safe.
</EvidencePin>

That is the architecture, end to end. You do not have to write it by hand.
