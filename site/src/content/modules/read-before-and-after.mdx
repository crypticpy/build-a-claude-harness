A note written and never read is just a file. What makes memory work is the second half: the
moment the saved note comes back into context. That moment is the next prompt.

<Fact>PreCompact writes the note; the next UserPromptSubmit reads it back. The pair is the Memento pattern.</Fact>

The write side, on PreCompact, distills the session before compaction would discard it. The
read side, on UserPromptSubmit, injects that note as the next turn begins. The read lives on
the prompt rather than only at startup for a precise reason: compaction can strike in the
middle of a long session, not just between sessions. Tie the read to startup alone and a
mid-session wipe would lose everything until the next launch. Tie it to the prompt and the
note returns on the very next turn, every time.

The read side injects the saved memory on UserPromptSubmit. Pairing it with the PreCompact
