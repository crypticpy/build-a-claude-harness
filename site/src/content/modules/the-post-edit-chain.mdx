Follow the post-edit branch down and you find three modules in a row. They are loaded
together but they run in sequence, and the sequence is deliberate: format, then log, then
hint.

<Fact>On every Write or Edit: format-lint runs first, rolling-log second, impact-hint last and best-effort.</Fact>

The formatter goes first so the file is in its final shape before anything records that the
edit happened. The logger goes second, stamping the operation against the settled file. The
impact hint goes last, because it is the riskiest and least essential of the three: a cheap
grep that warns you when you just edited a file other code imports. It is wrapped in its own
catch so that if it throws, the format and the log that already succeeded are untouched.

<EvidencePin file="reference/hooks/unified/unified-hook.mjs" lines={[112, 128]}>
The post-edit branch runs format, then log, then a hint guarded by its own try/catch. The
ordering and the isolation are both written here, on purpose.
</EvidencePin>

This is fail-silent at the level of a single branch: do the important work first, and fence
off the part most likely to break so it can only ever fail alone.
