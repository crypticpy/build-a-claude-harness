A hook runs inside your session, on the platform's clock. If a hook could throw and take
the turn down with it, every module would be a liability. So the harness makes one promise
and keeps it everywhere: the hook is invisible when it works and invisible when it fails.

<Fact>The dispatcher wraps every module in try/catch and exits zero no matter what happens inside.</Fact>

Three habits hold the promise together. The outer catch swallows any exception. The final
line is always a clean exit. And every dependency the harness might want, an API key, a
formatter, a git binary, is checked before use and skipped when absent. A missing tool
turns a feature off; it does not turn into a crash.

<EvidencePin file="reference/hooks/unified/unified-hook.mjs">
The dispatcher's outer try/catch and unconditional zero exit are the whole safety net. A
module is free to throw, because the boundary above it refuses to propagate the failure.
</EvidencePin>

<Caveat>Fail-silent is a reliability stance, not a correctness one. A hook that quietly does nothing is invisible by design, so when a feature seems dead, check stderr and the logs rather than expecting an error to surface on its own.</Caveat>
