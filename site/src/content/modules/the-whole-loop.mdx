Each room so far was one primitive in isolation. The capstone puts them on one track and runs
a token around it, so you can see how they connect. A turn enters at SessionStart, moves
through the prompt, fires tools that the log records, and approaches the edge of the context
window.

<Fact>The session is a closed loop. What is written before compaction is read back on the next turn in.</Fact>

The loop closes at the compaction boundary. PreCompact distills the turn into memory just
before the window collapses, and the next turn's UserPromptSubmit reads that memory back as
it begins. Without that pair, the line would be straight: a session would end and take its
context with it. With it, the track bends back on itself, and work resumes instead of
restarting.

<EvidencePin file="reference/settings.template.json">
Every station on the map is a real subscription in the settings file. The forward path and
the return path are both wired there, which is what lets the diagram draw a closed loop
rather than a dead end.
</EvidencePin>

That is the whole foundation. Phase 2 stops touring the primitives and starts taking the
reference harness apart, layer by layer, to answer the one question the loop was built for:
how does it run for hours without you?
