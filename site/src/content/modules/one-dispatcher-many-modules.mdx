Pull the dispatcher off the map and open it. Behind one entry point sit fifteen modules,
each a small file with a single job: format a file, append a log line, build the snapshot,
write the brain note. The dispatcher's whole role is to pick the right ones and stay out of
the way.

<Fact>Fifteen modules live behind one dispatcher, and each event imports only the ones its branch names.</Fact>

It does this lazily. The post-edit branch reaches for format-lint, rolling-log, and
impact-hint at the moment it runs, not before. A SessionStart event never imports the
formatter; a Stop event never imports the brain. So the cost of having fifteen modules is
close to zero until an event actually calls one, which is what lets the harness keep growing
modules without making any single event slower.

<EvidencePin file="reference/hooks/unified/unified-hook.mjs">
Each branch loads its modules on demand inside the switch. The fifteen modules impose no
shared startup cost, because importing them is deferred to the exact event that needs them.
</EvidencePin>
