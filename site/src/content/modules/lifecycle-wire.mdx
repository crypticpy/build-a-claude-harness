Picture the session as moving along a line, left to right, stopping at six points.
Claude Code already knows how to pause at each one on its own. What the harness adds
is what happens there.

This harness fills five of the six stops. Most filled stops run a single helper. One
stop is busier than the rest. When a tool just finished, two helpers run, one after the
other. First a helper tidies up the file that changed. Then a second helper writes a
short note that the tool ran. Tidy first, record second. The order is on purpose,
because the note should describe the file in its finished, cleaned-up state.

<Fact>This harness attaches a helper at five of the six moments. The moment right before a tool runs is left empty.</Fact>

The one empty moment is the spot right before a tool runs. It is a real moment, and
every session passes through it, but this harness chooses to do nothing there. So in the
picture it stays a hollow, dashed outline. The moment is real, but nothing is attached
to it.

<Caveat>These six moments belong to Claude Code itself, not to this harness. A future version of Claude Code could add a moment, rename one, or drop one. What is shown here is true for the version this site was built from, and no further.</Caveat>
