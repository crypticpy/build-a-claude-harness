You met the brain in Phase 1 as the one tier that pays. Here is the part the diagram left
out: it does not always pay. Before it spends a token, PreCompact checks whether the session
was substantial enough to be worth summarizing.

<Fact>PreCompact returns without an LLM call when the session has fewer than five tool calls.</Fact>

The threshold is a constant, five, and the logic is plain: a session where almost nothing
happened has almost nothing to distill, and a summary of nothing is not worth the call. So
the brain is gated twice over. It runs only on PreCompact, which is already rare, and even
then only when the session crossed the floor. That is how a feature that calls a model stays
genuinely cheap across a long day of work.

A constant sets the minimum tool calls, and the module returns early when the session falls
below it. The one billed operation in the whole harness will not fire for a trivial session.
</EvidencePin>
