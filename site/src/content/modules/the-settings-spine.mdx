Phase 1 toured the primitives. Phase 2 takes the reference harness apart, and the first
thing you find when you pry it open is that there is no spine to speak of, just a settings
file. Every behavior you watched is one subscription in it: an event, a matcher, a command.

<Fact>The settings file uses a literal $HOME everywhere. The installer copies it verbatim and never expands the path.</Fact>

That literal token is the portability trick. A path like `$HOME/.claude/hooks/unified-hook.mjs`
means nothing until Claude Code runs the hook and expands `$HOME` against the current
machine. Copy the file to a new laptop and it just works, because the token resolves to that
laptop's home. Expand it at install time and you have frozen one person's directory into a
file meant to travel.

<EvidencePin file="reference/settings.template.json">
Every path in the template is written with a literal $HOME. This is the file the installer
copies without substitution, which is what lets one settings file serve every machine.
</EvidencePin>
