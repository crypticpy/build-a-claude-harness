Memory sounds expensive. If the agent has to stay oriented across sessions, surely it is
calling a model constantly. It is not. Three of the four tiers never touch the LLM at all.

<Fact>The index, the rolling log, and the distilled query are free. Only the brain, at PreCompact, pays.</Fact>

The static index is built once at session start. The rolling log appends a line per tool
call. The distilled query is plain string truncation on read. None of that is billed. The
fourth tier, the brain, reads the whole log at compaction and distills it into a
cross-session note. That single call is the only place the money goes.

<EvidencePin file="reference/hooks/unified/modules/precompact-llm.mjs">
The PreCompact module makes one LLM call to write narrative memory and a lessons file. It is
the sole billed operation in the memory stack; every other tier is string work.
</EvidencePin>

Because the cost lives in one rare call, you can budget it directly. If you decide how many
tokens a session may spend on memory, that divided by the per-distillation cost is how many
times the brain may run.

<Caveat>The 8000-token figure is illustrative, chosen to make the arithmetic legible. The fact that survives is the topology: one billed call, run once per compaction, never on the hot path.</Caveat>
